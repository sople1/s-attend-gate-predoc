# 수동 출석 사용자 행동 시나리오

## 시나리오 개요

자동 출석이 실패한 상황에서 사용자들이 보이는 다양한 대체 방법 시도 행동을 기록한 시나리오입니다.
기술적 구현 세부사항은 system-scenarios로 분리됩니다.

---

## 시나리오 1: QR코드 스캔 대체 행동

### 사용자 프로필
- Sarah (35세, 마케팅 매니저)
- 중간 수준 기술 이해도
- 문제 해결 지향적 성향

### 행동 패턴
```
사용자 행동:
1. 자동 출석 실패 인지
2. 화면에 나타난 QR코드 옵션 확인
3. QR코드 스캔 모드로 전환
4. 입구 주변에서 QR코드 위치 탐색
5. 카메라로 QR코드 스캔 시도
6. 스캔 결과 확인 및 출석 완료 검증

감정 변화:
- 실패 인지: 약간의 당황
- 대안 발견: 문제 해결 의지
- 스캔 시도: 집중, 신중함
- 성공 시: 만족, 안도
```

- 대기 중: 호기심, 탐색욕

## 대체 방법별 사용자 선호도 분석

### QR코드 스캔 선호 그룹
**특징:**
- 시각적 피드백 선호
- 즉시 결과 확인 욕구
- 스마트폰 카메라 기능에 익숙

**행동 패턴:**
- 빠른 스캔 시도
- 각도와 거리 조절 실험
- 여러 QR코드 비교 시도

### 수동 입력 선호 그룹
**특징:**
- 정확한 통제 선호
- 단계별 확인 중시
- 텍스트 입력에 익숙

**행동 패턴:**
- 정보 정확성 반복 확인
- 입력 완료 전 재검토
- 오타나 실수 방지 주의

### 인적 지원 선호 그룹
**특징:**
- 대인 상호작용 편안함
- 복잡한 기술 과정 회피
- 확실한 해결 추구

**행동 패턴:**
- 도움 요청 시 상황 상세 설명
- 처리 과정 확인 및 이해
- 감사 표현과 피드백 제공

## 실패 상황별 대응 행동

### 1차 실패 (자동 출석 불가) 대응
**즉시 대응형:**
- 바로 QR코드 스캔 시도
- 문제 해결 의지 강함
- 추가 방법 적극 탐색

**신중 대응형:**
- 상황 파악 후 대안 선택
- 안전한 방법 우선 고려
- 확실한 해결책 추구

### 2차 실패 (QR코드 불가) 대응
**지속 시도형:**
- 다른 QR코드 탐색
- 스캔 방법 변경 시도
- 환경 변경 후 재시도

**대안 모색형:**
- 수동 입력 방법 탐색
- 관리자 도움 요청
- 완전히 다른 접근 방식

### 3차 실패 (모든 방법 불가) 대응
**포기 수용형:**
- 현 상황 받아들임
- 사후 처리 방법 문의
- 차선책 만족

**끝까지 시도형:**
- 관리자 직접 방문
- 특별 처리 요청
- 문제 원인 파악 시도

## 성공적인 수동 출석의 공통 요소

### 사용자 행동 특성
1. **단계적 접근**: 쉬운 방법부터 복잡한 방법 순서로 시도
2. **상황 적응**: 환경과 조건에 맞는 방법 선택
3. **결과 확인**: 각 단계별 성공/실패 명확히 확인
4. **도움 활용**: 필요시 주저 없이 지원 요청

### 만족도 결정 요인
1. **처리 속도**: 빠른 대안 제시와 처리
2. **명확성**: 각 방법의 설명과 안내
3. **신뢰성**: 선택한 방법의 성공률
4. **지원 품질**: 도움 요청 시 응답 수준

### 학습 효과
1. **방법 기억**: 성공한 방법을 다음에 우선 사용
2. **환경 인식**: 특정 상황에서의 최적 방법 학습
3. **예방 조치**: 미리 대비책 준비
4. **타인 도움**: 경험 공유로 다른 사용자 지원

---

## 개선 방향 제안

### 사용자 경험 최적화
1. **방법 순서**: 사용자 성향별 추천 순서 제공
2. **학습 반영**: 이전 성공 방법 우선 제안
3. **상황 인식**: 환경에 따른 최적 방법 자동 제안
4. **피드백 활용**: 실패 원인 분석으로 개선점 도출

### 지원 시스템 강화
1. **즉시 도움**: 실시간 문제 해결 지원
2. **셀프 가이드**: 단계별 자가 해결 안내
3. **상황별 대응**: 다양한 실패 상황별 맞춤 해결책
4. **사후 관리**: 문제 해결 후 만족도 확인 및 개선
