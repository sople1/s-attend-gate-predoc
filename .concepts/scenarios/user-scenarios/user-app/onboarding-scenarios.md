# User App - 온보딩 시나리오

## 온보딩 개요

참가자가 앱을 설치하고 행사에 참가하기까지의 전체 과정을 관리합니다.
4개 독립 서비스 아키텍처에서 Event Management와 토큰 기반으로 연동됩니다.

## 사전 준비 단계

### 1. 행사 등록 완료 (D-7일 ~ D-1일)

```mermaid
sequenceDiagram
    participant EM as Event Management
    participant Email as 이메일 시스템  
    participant User as 참가자
    participant UA as User App
    
    EM->>Email: 참가자 토큰 생성 및 이메일 발송
    Email->>User: 앱 설치 안내 + 토큰 링크
    User->>UA: 앱 다운로드 및 설치
    User->>UA: 토큰 링크 클릭 또는 수동 입력
    UA->>EM: 토큰 검증 요청
    EM->>UA: 행사 정보 및 권한 전송
    UA->>User: 행사 추가 완료 알림
```

```

1. 권한 요청 타이밍
   - A: 앱 시작 즉시
   - B: 기능 사용 시점

2. 토큰 입력 방식
   - A: 수동 타이핑
   - B: 카메라 OCR 인식

3. 온보딩 길이
   - A: 상세한 5단계
   - B: 간소한 3단계

측정 지표:
- 완료율
- 완료 시간
- 첫 성공적 출석까지 시간
```
