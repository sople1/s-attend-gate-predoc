# 수동 출입 처리 및 예외 상황 대응 시나리오

## 🎯 개요

BLE 자동 감지나 QR 스캔이 실패했을 때, 또는 특별한 상황에서 현장 스태프가 수동으로 출입을 처리하는 시나리오입니다.

---

## 🔧 시나리오 1: 기본 수동 출입 처리

### 1.1 수동 처리 모드 진입

**사용자**: 현장 보안 요원  
**상황**: 참가자의 BLE/QR 출석 체크 실패  
**목표**: 신속한 수동 출입 처리

#### 사용자 여정

```
1. 수동 처리 요청 접수
   참가자: "앱이 작동하지 않아요"
   요원: "잠시만 기다려 주세요, 확인해드릴게요"

2. 태블릿에서 "수동 처리" 모드 진입
   ┌─────────────────────────────────┐
   │ 🔧 수동 출입 처리               │
   │                                 │
   │ 📋 참가자 정보 확인 방법:       │
   │ [🔍 이름/연락처 검색]           │
   │ [📱 QR 코드 스캔]              │
   │ [💳 참가자 ID 입력]            │
   │ [📧 이메일 검색]               │
   │                                 │
   │ 💡 최근 처리 내역:              │
   │ • 김철수 (3분 전)              │
   │ • 이영희 (7분 전)              │
   └─────────────────────────────────┘

3. 참가자 정보 검색 및 확인
   검색 결과:
   ┌─────────────────────────────────┐
   │ 👤 참가자 정보                  │
   │ 이름: 박민수                    │
   │ ID: DEV2024-001234             │
   │ 이메일: park@example.com       │
   │ 연락처: 010-1234-5678          │
   │                                 │
   │ 🎫 등록 정보:                   │
   │ 행사: Developer Conference     │
   │ 티켓: 일반 참가자               │
   │ 상태: ✅ 등록 완료              │
   │                                 │
   │ 📋 출입 이력:                   │
   │ • 없음 (첫 출입)               │
   │                                 │
   │ [✅ 출입 승인] [❌ 출입 거부]    │
   └─────────────────────────────────┘

## 🚨 시나리오 2: 예외 상황 대응

### 2.1 의심스러운 참가자 대응

**상황**: 등록되지 않았거나 의심스러운 참가자 접근  
**목표**: 보안을 유지하면서 적절한 대응 조치

#### 대응 프로세스

```
1. 초기 상황 파악
   참가자: "등록했는데 시스템에 없다고 나와요"
   
2. 상세 확인 절차
   ┌─────────────────────────────────┐
   │ 🔍 미등록 참가자 확인           │
   │                                 │
   │ 📋 확인 항목:                   │
   │ ☐ 이름 및 신분증 확인           │
   │ ☐ 등록 이메일 확인              │
   │ ☐ 결제 내역 확인               │
   │ ☐ 등록 확인서 확인              │
   │                                 │
   │ 📱 추가 확인:                   │
   │ ☐ 등록 시스템 재검색            │
   │ ☐ 유사 이름 검색               │
   │ ☐ 이메일 도메인 검색            │
   │                                 │
   │ 🎯 처리 방안:                   │
   │ [📞 등록팀 문의]               │
   │ [📋 임시 등록]                 │
   │ [❌ 출입 거부]                 │
   │ [👮 보안팀 호출]               │
   └─────────────────────────────────┘

3. 처리 결과별 대응
   
   Case A: 등록 확인됨
   ✅ "확인되었습니다. 시스템 동기화 문제였네요"
   → 임시 출입 승인 + 시스템 업데이트 요청
   
   Case B: 등록되지 않음
   ⚠️ "등록 내역을 찾을 수 없습니다"
   → 등록팀 연락 + 임시 대기 요청
   
   Case C: 의심스러운 상황
   🚨 "추가 확인이 필요합니다"
   → 보안팀 호출 + 상황 기록
```

2. 오프라인 임시 운영
   ┌─────────────────────────────────┐

   │                                 │

   │ 1. 참가자 신분증 확인           │
   │ 2. 종이 명단 대조               │
   │ 3. 수기 출입 기록               │
   │ 4. 복구 후 일괄 입력            │
   │                                 │
   │ 📞 긴급 연락:                   │
   │ [🔧 기술팀] [👮 보안팀] [📋 관리자] │
   └─────────────────────────────────┘

3. 종이 기록 양식
   
   비상 출입 기록부
   ─────────────────────────────────
   날짜: 2024-07-15    게이트: A

   담당자 서명: ___________

4. 시스템 복구 후 일괄 처리
   복구 완료 ✅
   
   일괄 데이터 입력:
   📋 종이 기록 23건 → 시스템 입력
   ⏱️ 예상 소요시간: 15분
   
   검증 절차:
   ✅ 중복 출입 체크
   ✅ 등록 상태 재확인
   ✅ 시간 정보 검증
   ✅ 처리 완료 확인
```
